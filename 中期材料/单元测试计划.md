# 作业OCR自动批改系统 — 单元测试计划

## 1. 引言

### 1.1 测试目的

验证系统各模块的内部函数和方法在不同输入条件下的正确性，确保每个最小可测试单元按预期工作。

### 1.2 测试范围

| 模块 | 文件 | 测试重点 |
|------|------|---------|
| 数据模型 | `backend/models/question.py` | 数据类创建、枚举、属性计算 |
| 结果模型 | `backend/models/result.py` | GradingReport汇总计算 |
| OCR引擎 | `backend/core/ocr_engine.py` | 识别接口、置信度过滤、排序 |
| 图像预处理 | `backend/core/preprocessor.py` | 各处理步骤输入输出 |
| 文本解析 | `backend/core/parser.py` | 题号提取、答案拼接 |
| 批改引擎 | `backend/core/grader.py` | 三种题型评分逻辑 |
| 报告导出 | `backend/core/exporter.py` | CSV/HTML格式正确性 |
| 数据库操作 | `backend/database.py` | CRUD操作、查询过滤 |
| API客户端 | `frontend/api_client.py` | 请求构造、响应处理 |

### 1.3 测试框架

- **pytest** — 测试执行框架
- **unittest.mock** — Mock外部依赖（PaddleOCR、数据库等）
