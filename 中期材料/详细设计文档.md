### 2.2 OCR引擎模块 (`backend/core/ocr_engine.py`)

#### 2.2.1 模块职责

封装PaddleOCR，提供统一的文字识别接口。

#### 2.2.2 函数设计

**`get_ocr() → PaddleOCR`**

- 功能：获取PaddleOCR单例实例
- 实现：懒加载模式，首次调用时初始化
- 配置参数：
  - `lang='ch'`（中文）
  - `det_model_dir`：本地检测模型路径
  - `rec_model_dir`：本地识别模型路径
  - `enable_doc_orientation_classify=False`
  - `enable_doc_unwarping=False`
  - `enable_textline_orientation=False`

**`recognize(image: Union[str, np.ndarray]) → List[OcrResult]`**

- 功能：对输入图片执行OCR识别
- 输入：图片文件路径或numpy数组
- 输出：按位置排序的OcrResult列表
- 处理流程：
  1. 调用PaddleOCR的predict方法
  2. 遍历识别结果，过滤低置信度结果 (< `OCR_CONFIDENCE_THRESHOLD`)
  3. 提取bbox、text、confidence构造OcrResult对象
  4. 按y坐标排序（从上到下），y坐标相近时按x坐标排序（从左到右）
- 异常处理：捕获所有异常，返回空列表

### 2.3 图像预处理模块 (`backend/core/preprocessor.py`)

#### 2.3.1 模块职责

对作业图片进行增强处理，提高OCR识别准确率。

#### 2.3.2 函数设计

**`preprocess(image_path: str) → np.ndarray`**

处理流水线：

```
原始图片 → 灰度化 → 去噪 → 纠偏 → 二值化 → 输出
```

**`to_grayscale(img: np.ndarray) → np.ndarray`**

- 功能：BGR彩色图像转灰度图
- 算法：`cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)`

**`denoise(gray: np.ndarray) → np.ndarray`**

- 功能：去除图像噪声
- 算法：非局部均值去噪 `cv2.fastNlMeansDenoising`
- 参数：`h=10, templateWindowSize=7, searchWindowSize=21`

**`deskew(gray: np.ndarray) → np.ndarray`**

- 功能：检测并修正图像倾斜
- 算法流程：
  1. Canny边缘检测
  2. 霍夫直线变换检测直线
  3. 计算各直线角度，取中位数
  4. 若角度 > 0.5°，进行仿射旋转修正

**`binarize(gray: np.ndarray) → np.ndarray`**

- 功能：将灰度图转为黑白二值图
- 算法：自适应高斯阈值 `cv2.adaptiveThreshold`
- 参数：`blockSize=11, C=2`

### 2.4 文本解析模块 (`backend/core/parser.py`)

#### 2.4.1 模块职责

将OCR识别出的文本列表解析为"题号→答案"的映射关系。

#### 2.4.2 函数设计

**`parse_answers(ocr_results: List[OcrResult]) → Dict[int, str]`**

- 输入：排序后的OCR识别结果列表
- 输出：`{题号: 答案文本}` 字典
- 解析算法：
  1. 对每条OCR文本进行正则匹配，提取题号
  2. 支持的题号格式：
     - `(1)`, `（1）` — 中英文括号
     - `第1题`
     - `例1`
     - `1.`, `1、`, `1)`
  3. 匹配成功：记录题号和对应答案文本
  4. 匹配失败：将文本追加到上一个题号的答案中（多行答案支持）

**`grade_multiple_choice(recognized: str, question: Question) → Tuple[bool, float]`**

- 评分算法：
  1. 从识别文本中提取选项字母 (A-F)
  2. 精确匹配：完全一致 → (True, 1.0)
  3. 部分匹配：选择的为正确答案的子集且无错选 → (True, 0.5)
  4. 否则 → (False, 0.0)

**`grade_calculation(recognized: str, question: Question) → Tuple[bool, float]`**

- 评分算法：
  1. 尝试从文本中提取数值（支持 `结果=数值` 格式）
  2. 尝试直接数值解析
  3. 尝试安全表达式求值（仅支持 +, -, ×, ÷）
  4. 数值比较：容差 0.1% (`CALCULATION_TOLERANCE = 0.001`)
  5. 回退：字符串比较

### 2.8 API路由层 (`backend/app.py`)

#### 2.8.1 POST `/api/upload`

```
请求：multipart/form-data, file字段
处理：
  1. 验证文件类型（白名单）
  2. 生成UUID作为file_id
  3. 重命名文件存储到uploads/目录
  4. 保存记录到homeworks表
响应：{"file_id": "uuid", "filename": "...", "message": "上传成功"}
```

#### 2.8.2 POST `/api/grade`

```
请求：
{
  "file_id": "uuid",
  "questions": [
    {"number": 1, "type": "fill_blank", "answer": "北京", "points": 2.0, "alternatives": []}
  ]
}
处理：
  1. 根据file_id查找图片
  2. 图像预处理
  3. OCR识别
  4. 解析OCR结果
  5. 构造Question对象列表
  6. 调用批改引擎评分
  7. 结果存入数据库
响应：
{
  "results": [...],
  "summary": {"total_points": x, "earned_points": y, "percentage": z},
  "ocr_count": n,
  "ocr_results": [...],
  "parsed_answers": {...},
  "grading_id": id
}
```

#### 2.8.3 GET `/api/history`

```
请求参数：keyword, date_from, date_to, min_score, max_score, page, per_page
处理：调用query_history()进行条件查询与分页
响应：
{
  "total": n,
  "page": p,
  "per_page": pp,
  "records": [...]
}
```

#### 3.2.2 GradeWorker 线程类

- 继承 `QThread`
- 在后台线程执行批改API调用
- 通过信号通知主线程：
  - `finished(dict)`：批改完成，传递结果
  - `error(str)`：批改出错，传递错误信息

#### 3.2.3 主要交互流程

1. **打开图片**：触发文件对话框 → 选择图片 → ImagePanel显示 → 上传后端获取file_id
2. **开始批改**：收集AnswerPanel的题目数据 → 启动GradeWorker → 禁用按钮 → 等待回调
3. **批改完成**：ResultPanel展示结果 → ImagePanel显示OCR标注 → 状态栏更新
4. **导出报告**：触发保存对话框 → 调用导出API → 写入本地文件

## 5. 错误处理设计

### 5.1 后端错误处理

| 场景 | 处理方式 |
|------|---------|
| 文件未上传 | 返回 400，提示"没有文件" |
| 文件类型不允许 | 返回 400，提示"不支持的文件格式" |
| file_id不存在 | 返回 404，提示"文件不存在" |
| OCR识别失败 | 返回 500，记录日志 |
| 数据库操作失败 | 返回 500，回滚事务 |

### 5.2 前端错误处理

| 场景 | 处理方式 |
|------|---------|
| 后端未启动 | 弹出提示"无法连接后端服务" |
| 上传失败 | 状态栏显示错误信息 |
| 批改失败 | GradeWorker发出error信号，弹出错误对话框 |
| 网络超时 | requests超时设置，弹出提示 |
